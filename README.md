# 项目技术栈与特点

## 项目概述
黑马点评是一个基于Spring Boot开发的O2O（Online to Offline）点评平台，集成了商户管理、用户社交、优惠券秒杀等功能，采用前后端分离架构。

## 技术栈

### 后端框架
- **Spring Boot 2.3.12.RELEASE** - 核心框架
- **Spring MVC** - Web层框架
- **Spring AOP** - 面向切面编程
- **MyBatis-Plus 3.4.3** - ORM框架，简化数据库操作

### 数据存储
- **MySQL 5.1.47** - 关系型数据库
- **Redis** - 缓存数据库，使用Lettuce连接池
- **Redisson 3.13.6** - Redis分布式锁客户端

### 工具库
- **Hutool 5.7.17** - Java工具类库
- **Lombok** - 简化实体类代码
- **AspectJ** - AOP支持

### 开发环境
- **Java 1.8** - 开发语言
- **Maven** - 项目构建工具

## 核心功能模块

### 1. 用户认证与授权
#### 技术实现
- **基于Redis的Token存储**：用户登录后生成Token，存储在Redis Hash结构中
- **Token自动刷新机制**：每次请求自动刷新Token有效期
- **拦截器链**：
  - `RefreshTokenInterceptor`（优先级0）：刷新Token，存储用户信息到ThreadLocal
  - `LoginInterceptor`（优先级1）：验证登录状态
- **ThreadLocal**：线程隔离存储用户信息，避免频繁查询Redis

#### 优势
- 无状态认证，支持集群部署
- Token自动续期，提升用户体验
- 双层拦截器设计，职责分离清晰

### 2. 缓存策略
#### 技术实现
封装了`CacheClient`工具类，提供多种缓存策略：

**缓存穿透解决**
- 空值缓存：将不存在的数据缓存为空字符串，设置较短TTL
- 缓存空值标识：区分"缓存不存在"和"数据不存在"

**缓存击穿解决**
- **互斥锁方案**：使用Redis `setIfAbsent` 实现分布式锁，防止大量请求同时查询数据库
- **逻辑过期方案**：缓存数据包含逻辑过期时间，过期后异步重建，不阻塞请求

**缓存雪崩预防**
- 随机TTL：避免大量缓存同时过期

#### 优势
- 多种策略可选，适应不同场景
- 异步缓存重建，提升系统吞吐量
- 统一的缓存工具类，代码复用性高

### 3. 分布式锁
#### 技术实现
**自定义SimpleRedisLock**
- 基于`setIfAbsent`实现
- 使用Lua脚本保证锁释放的原子性
- UUID + 线程ID作为锁标识，防止误删

**Redisson分布式锁**
- 看门狗机制：自动续期，防止业务执行时间超过锁TTL
- 可重入锁：支持同一线程多次获取锁
- 更完善的异常处理

#### 应用场景
- 缓存重建时的互斥访问
- 秒杀业务的一人一单限制
- 库存扣减的并发控制

#### 优势
- 集群环境下保证互斥性
- 自动续期，避免锁过期
- Lua脚本保证原子性

### 4. 分布式ID生成器
#### 技术实现
**RedisIdWorker**
- 基于Redis的`INCR`命令实现序列号自增
- 按天分片：`icr:order:2022:01:01`
- ID结构：`时间戳（32位）| 序列号（32位）`
- 起始时间戳：2022-01-01 00:00:00

#### 优势
- 全局唯一，不依赖数据库
- 时间有序，便于分页查询
- 高性能，Redis原子操作
- 可扩展，支持多业务场景

### 5. 社交功能
#### 技术实现
**关注/取关**
- 数据库存储关注关系
- Redis Set存储关注列表，提升查询性能

**共同关注**
- 使用Redis Set的`intersect`命令求交集
- 时间复杂度O(N)，性能优于数据库JOIN

**博客点赞**
- 使用Redis ZSet存储点赞用户
- Score为时间戳，支持按时间排序
- 数据库存储点赞总数

**Feed流推送（推模式）**
- 用户发布笔记时，推送到所有粉丝的收件箱
- 使用Redis ZSet存储，Score为发布时间
- 滚动分页查询：基于分数（时间戳）和偏移量

#### 优势
- 高性能：利用Redis数据结构特性
- 实时性好：推模式保证粉丝及时收到更新
- 可扩展：支持海量用户和笔记

### 6. 秒杀功能
#### 技术实现
**库存扣减**
- 使用MyBatis-Plus的条件更新器
- `gt("stock", 0)` 防止超卖
- CAS思想，乐观锁

**一人一单限制**
- 数据库查询校验：`query().eq("user_id", userId).eq("voucher_id", voucherId).count()`
- 分布式锁：基于用户ID加锁，防止并发下单

**事务管理**
- `@Transactional` 注解保证数据一致性
- 使用AOP代理对象调用事务方法

#### 优势
- 防止超卖：数据库层面保证
- 并发安全：分布式锁保证
- 数据一致：事务保证

### 7. 数据库设计
#### 核心表结构
- **tb_user**：用户基本信息
- **tb_user_info**：用户详细信息
- **tb_shop**：商户信息
- **tb_shop_type**：商户类型
- **tb_voucher**：优惠券信息
- **tb_seckill_voucher**：秒杀优惠券
- **tb_voucher_order**：优惠券订单
- **tb_blog**：探店笔记
- **tb_blog_comments**：笔记评论
- **tb_follow**：关注关系

#### 设计特点
- 用户信息拆分：基本信息和详细信息分离
- 优惠券秒杀独立表：支持普通优惠券和秒杀优惠券
- 社交功能独立表：关注、点赞、评论

## 项目优势特点

### 1. 高性能
- **多级缓存**：Redis缓存 + 数据库，减少数据库压力
- **异步处理**：缓存重建异步执行，不阻塞请求
- **连接池**：Redis连接池优化，提升并发能力

### 2. 高可用
- **分布式锁**：集群环境下保证数据一致性
- **Token自动刷新**：提升用户体验，减少重新登录
- **异常处理**：全局异常处理，统一错误响应

### 3. 高并发
- **乐观锁**：库存扣减使用CAS，避免悲观锁性能问题
- **分布式锁**：Redisson看门狗机制，支持长业务
- **ID生成器**：高性能分布式ID生成，避免数据库瓶颈

### 4. 可扩展
- **前后端分离**：前端独立部署，支持多端适配
- **微服务友好**：模块化设计，易于拆分为微服务
- **Redis数据结构**：充分利用Redis特性，易于扩展新功能

### 5. 代码质量
- **统一响应**：Result统一封装响应数据
- **工具类封装**：CacheClient、RedisIdWorker等工具类，提升复用性
- **AOP应用**：代理对象调用，解决事务失效问题
- **代码注释**：关键逻辑有详细注释

### 6. 安全性
- **Token认证**：无状态认证，支持集群
- **密码加密**：使用PasswordEncoder加密存储
- **SQL注入防护**：MyBatis-Plus参数化查询
- **XSS防护**：前后端数据校验

## 技术亮点

### 1. 缓存三兄弟解决方案
- **缓存穿透**：空值缓存 + 布隆过滤器（可扩展）
- **缓存击穿**：互斥锁 + 逻辑过期
- **缓存雪崩**：随机TTL + 缓存预热

### 2. 分布式锁双重实现
- 自定义SimpleRedisLock：轻量级，适合简单场景
- Redisson：功能完善，适合复杂场景

### 3. Feed流推拉结合
- 推模式：粉丝量少时使用
- 拉模式：粉丝量多时使用（可扩展）
- 滚动分页：基于分数的高效分页

### 4. 秒杀系统设计
- 库存预热：提前加载到缓存
- 异步下单：消息队列解耦（可扩展）
- 限流降级：保护系统稳定性（可扩展）

## 可优化方向

### 1. 数据库优化
- 添加唯一索引：`tb_voucher_order`表添加`(user_id, voucher_id)`唯一索引
- 读写分离：主从复制，读写分离
- 分库分表：垂直拆分 + 水平拆分

### 2. 缓存优化
- 缓存预热：系统启动时加载热点数据
- 多级缓存：本地缓存 + Redis缓存
- 缓存一致性：Canal监听数据库变更

### 3. 秒杀优化
- 消息队列：RabbitMQ/Kafka异步下单
- 限流：Redis + 令牌桶算法
- 降级：关闭非核心功能

### 4. 监控告警
- 接口监控：Prometheus + Grafana
- 日志收集：ELK Stack
- 链路追踪：SkyWalking

## 总结
项目是一个典型的互联网应用，涵盖了用户认证、缓存策略、分布式锁、社交功能、秒杀系统等核心技术。项目架构清晰，代码规范，技术选型合理，是一个优秀的Spring Boot实战项目。通过该项目可以学习到：
- Redis的高级应用
- 分布式锁的实现原理
- 缓存策略的设计思想
- 社交功能的技术实现
- 秒杀系统的架构设计

项目代码具有良好的扩展性，可以根据业务需求进行优化和升级。
